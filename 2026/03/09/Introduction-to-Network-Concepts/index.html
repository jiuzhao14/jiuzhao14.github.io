技术文档：IEEE 802.1Q Priority (PCP/CoS) 详解

文档版本: 1.0
日期: 2026-03-09

1. 概述 (Overview)

IEEE 802.1Q Priority，通常被称为 CoS (Class of Service) 或历史上的 802.1p，是一种在以太网（Layer 2）层面上实现服务质量（QoS）的机制。

它利用 802.1Q VLAN 标签头中的特定字段，允许网络设备（交换机、网卡）对数据帧进行分类和优先级排序。这确保了在网络拥塞时，关键业务流量（如语音、视频、网络控制信令）能获得优先转发。

2. 协议背景与标准 (Background)

  • 标准归属: 最初由 IEEE 802.1p 定义，后被整合进 IEEE 802.1Q 标准（VLAN Tagging）。
  • 工作层级: OSI 模型第 2 层（数据链路层）。
  • 核心概念: PCP (Priority Code Point)。这是 802.1Q 标签中用于表示优先级的 3 位字段。

3. 帧结构分析 (Frame Structure)

在标准的以太网帧中插入 4 字节的 802.1Q Tag 后，帧结构如下：

TCI (Tag Control Information) 字段详解

TCI 包含三个子字段，其中 PCP 是本文档的核心：

  1. PCP (Priority Code Point):
    • 长度: 3 bits
    • 取值范围: 0 - 7 (二进制 000 - 111)
    • 作用: 定义帧的优先级。值越大，优先级通常越高（除了 0 的特殊情况，见下文）。
  2. DEI (Drop Eligible Indicator): 1 bit，用于指示在拥塞时该帧是否可被丢弃（原 CFI 位）。
  3. VID (VLAN Identifier): 12 bits，标识 VLAN ID (0-4095)。

4. 优先级定义 (Priority Levels)

根据 IEEE 802.1Q 标准推荐，3 位 PCP 值的 8 种状态映射如下：

PCP 值 (二进制) 优先级名称 流量类型推荐 (Traffic Type) 典型应用场景
7 (111) Network Control 网络控制 STP, LACP, 路由协议 (最高优先级)
6 (110) Internetwork Control 网络互联控制 必须保障的网络状态同步流量
5 (101) Voice 语音 VoIP (对延迟和抖动极度敏感, <10ms)
4 (100) Video 视频 视频会议, 监控流 (高带宽, 低延迟)
3 (011) Critical Application 关键应用 数据库同步, 关键业务系统
2 (010) Excellent Effort 尽力而为(优) 重要的普通业务 (CEO 直播等)
0 (000) Best Effort (Default) 尽力而为(默认) 普通上网, 文件传输 (默认优先级)
1 (001) Background 背景流 备份数据, 批量传输 (最低优先级)

注意: 虽然数值上 1 > 0，但在标准定义中，0 (Best Effort) 的优先级通常高于 1 (Background)。背景流量被设计为仅在网络空闲时传输。

5. 工作机制 (Working Mechanism)

802.1Q Priority 仅在发生网络拥塞时生效。其处理流程如下：

5.1 入口分类 (Ingress Classification)

当交换机端口接收到数据帧时：

  • Trust Mode (信任模式): 交换机信任帧中携带的 PCP 值，直接用于内部处理。
  • Re-marking (重标记): 交换机忽略原有 PCP，根据 ACL 或端口策略强制修改 PCP 值。

5.2 队列映射 (Queue Mapping)

交换机硬件通常只有 4 或 8 个物理出口队列（Egress Queues）。

  • 映射关系: 8 个 PCP 值必须映射到有限的硬件队列中。
  • 示例 (4队列交换机):
    • Queue 3 (High): PCP 7, 6, 5
    • Queue 2: PCP 4, 3
    • Queue 1: PCP 2, 0
    • Queue 0 (Low): PCP 1

5.3 出口调度 (Egress Scheduling)

当多个队列都有数据等待发送时，调度算法决定谁先通过：

  • SP (Strict Priority): 严格优先级。只要高优先级队列有数据，低优先级永远不发（可能导致低优先级“饿死”）。通常用于 Voice (PCP 5)。
  • WRR (Weighted Round Robin): 加权轮询。按权重比例发送，例如高优先级占 50% 带宽，低优先级占 10%。
  • 混合模式: 队列 3 (Voice) 使用 SP，其他队列使用 WRR。

6. 与 Layer 3 QoS (DSCP) 的关系

  • Layer 2 (802.1Q PCP): 仅在以太网链路上有效。经过路由器（Layer 3 转发）时，VLAN Tag 会被剥离，PCP 信息丢失。
  • Layer 3 (IP DSCP): 位于 IP 头部的 ToS 字段，端到端保留。

最佳实践: 在网络边界（交换机/路由器）配置 CoS-to-DSCP 映射。

  • 入站时：将 L2 PCP 映射为 L3 DSCP (例如 PCP 5 -> DSCP EF)。
  • 出站时：将 L3 DSCP 映射回 L2 PCP，以便下一跳交换机识别。

总结:
802.1Q Priority 是局域网内保障关键业务性能的基石。在开发涉及实时通信（如 VoIP、工控协议）的软件时，正确设置 Socket 优先级并配置 OS 的 VLAN 映射是必不可少的步骤。

Wireshark TCP Trace Graph Tutorial

Wireshark offers a couple of graphs for TCP analysis: RTT, throughput, window scaling, and the time sequence graphs.

Dark yellow ACK line

One line that is not found in the Stevens graph is the dark yellow ACK line. Here the ACK number of the other side, the receiver side, is displayed. This allows you to see when a sequence number has been confirmed.

If the green line is too far apart from the bule line, packets will not be acknowledged in time. If the green line does not rise, the transmitter waits for acknowledgments. This is symptomatic of disconnections or performance issues.

Out of Order or Retransmission

Here you see how the blue packets skip a sequence number and send it a moment later. This can be either an out-of-order or retransmission.

DUP ACKs can be recognized in the TCP trace graph because the ACK line does not increase, and small strokes below the line are visible. DUP acks are used in packet loss to trigger fast retransmissions. Too many DUP acks can be an indication of too many packet losses.

Selective ACKs

Selective ACKs are shown in red. Here you see how SACKs are sent for a short time, but within 100ms, the problem is solved by retransmitting the requested bytes.
