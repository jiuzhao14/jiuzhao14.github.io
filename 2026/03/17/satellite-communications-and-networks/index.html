Air Interface: The Physical Layer and Antennas

卫星连接的最底层是发射机与接收机之间的链路。在本章中，我们将探讨卫星系统所采用的数字通信信号、所需的差错控制策略以及用于通信的天线；此外，我们还将深入研究链路预算设计，其中包括理解发射机与接收机之间的信道特性，以及该信道对信号造成的劣化影响。

我们将探讨现代通信卫星中所采用的几种精选空中接口。其中包括宽带卫星系统的主力军——数字视频广播卫星（DVB-S）系列及其第二代扩展标准 DVB-S2X；一种被部分小型卫星用于物联网（IoT）通信的空中接口——LoRa；以及第五代移动通信网络（5G）的非地面网络（NTN）接口——该接口通过实现智能手机与卫星之间的直接连接，有望彻底变革消费者在日常生活中使用卫星系统的方式。在本章中，您将学到：

  • 数字通信信号是如何形成与调制的，以及卫星系统中采用了哪些调制方法。
  • 卫星通信系统中采用了哪些类型的差错控制策略？这些策略涵盖了自动重传请求（ARQ）、前向纠错（FEC）方法及其组合。
  • 天线技术基础，涵盖相控阵天线与多波束卫星。
  • 如何针对 GEO 和 LEO 系统计算链路预算，以确保终端用户及其应用获得理想的性能。
  • LEO 系统中的多普勒频移效应，以及如何在卫星端或终端端对其进行补偿。
  • 简要介绍所讨论的技术如何在当前现行的标准中得到应用，以及（例如在 5G 系统中）还采用了哪些额外的特性。

Digital Communication Signals

在我们日常生活中，所有的通信都是以信号的形式进行的。这些信号（例如人类语音）本质上属于模拟信号。尽管模拟信号长期以来一直被用于远距离通信，但它们容易受到失真影响，安全性较弱，且无法用于传输消费及工业应用领域所需的全部数字数据。数字通信是指通过点对点或点对多点信道，以数字比特流的形式进行数据的传输与接收。数字信号是原始数据或模拟信号经过数字化编码后的版本。这种方式克服了模拟传输所固有的上述弊端，从而实现了信号的清晰传输，使其不易受到噪声、干扰及失真的影响。数字化过程将连续的正弦波模拟信号转化为由“0”和“1”组成的离散信号。

数字通信系统的基本组成单元包括换能器和模数转换器（A/D转换器），用于将模拟信号转换为数字信号。根据奈奎斯特-香农（Nyquist-Shannon）定理，对于带宽为B的模拟信号，其采样率至少应达到每秒2B个样本，方能完整捕获该信号所承载的所有信息。若要研习数字信号理论与通信原理，Sklar (2001) 和 Proakis (2007) 是极佳的参考资料。图4.1展示了数字通信系统的一个简化框图。信源编码器将输入波形压缩为比特流，旨在高效利用有限的信道带宽。信道编码器则在经信源编码后的数据中加入冗余比特，以便接收端能够进行纠错处理。最后，待发送的信号先由载波进行调制，并通过数模转换器（D/A转换器）转换为模拟形式，随后经由天线发送至信道。

信号经由无线信道传输至接收端，在此处，信号被解调并重新转换为数字格式。信道解码器负责执行必要的纠错操作，并去除冗余比特，从而恢复原始信号。最后，信源解码器将这些比特转换为信号波形；该波形最终抵达目的地，并可在需要时被还原为模拟语音信号。上述编码器和解码器模块所处理的信号被称为基带信号，它们占据了电信网络中最低的频段。接下来，我们将探讨基带信号的基础知识，介绍卫星网络中典型的调制与编码方法；此外，我们还将深入研究旨在提升系统容量与可靠性的各类技术，其中包括自适应调制与编码（ACM）以及混合自动重传请求（HARQ）。

Network Layer and Transport Layer

网络层负责实现变长数据包从源端到目的端的传输，并在网络中执行路由功能。源端网络层的功能包括：接收来自上层（即传输层和应用层）的数据包，并将其封装成包含源地址和目的地址的网络层数据报。随后，网络层将该数据包交付给数据链路层，以便将其发送至接收端。目的端节点对接收到的数据报进行解封装，从中提取出原始数据包，并将其交付给相应的传输层。

传输层位于网络层与应用层之间，负责在源端与目的端之间建立端到端的连接。在发送端，传输层将来自应用层的数据进行分段处理，并在每个数据段的头部添加源端口号和目的端口号；随后，将这些数据段交付给网络层。在接收端，传输层接收来自网络层的数据，并通过相应的端口将这些分段数据重新组装并交付给应用层。
因此，在卫星通信系统中，网络层和传输层对于保障终端用户的服务质量至关重要。在本章中，您将学习以下主题：

(a) 网络技术基础，包括网络的主要功能及相关的性能指标。
(b) 流量测量与建模的重要性；网络参考模型（OSI 和 TCP/IP）以及各相关协议层的功能。
(c) 分组​​交换网络及 IP 协议的工作原理。
(d) 具体的卫星组网相关专题，例如星间互连、路由选择以及卫星间切换（切换）的实现机制。
(e) 什么是 QUIC 协议，以及相较于传统传输协议，它是如何提升网络性能的。
(f) 最后，我们将探讨软件定义网络（SDN）——作为一种面向未来的方法，它能够实现资源的端到端编排，并促成空中、空间及地面域各网络片段的无缝集成。

Network Functions and Network-Level Metrics

网络层主要具备三项功能或服务：即分组、路由和转发。所谓“分组”，是指将从上层接收到的数据在源端封装成数据包，并在目的端进行解封装。鉴于源端与目的端之间的网络中可能存在多条路径，因此需要利用“路由”功能来寻找最佳路径。路由是指将数据从特定设备传输至另一设备的过程；源端与目的端之间的传输路径可能包含多个“跳段”（hops）。处于该传输路径上的路由器，不得更改数据包的源地址或目的地址。路由协议的作用在于协助路由器之间相互协调，在确保数据包及时送达的同时，避免网络拥塞。最后，“转发”是指当数据包抵达路由器的接口时，路由器所执行的操作：它通过某个出接口将数据包转发至特定的相连网络（单播），或转发至多个网络（多播）。

因此，卫星系统中网络层的目标在于：在终端之间建立连接，通过相关的卫星链路和地面链路对信号进行路由，并确保满足目标应用所需的业务质量（QoS）要求。网络中的一条链路包含两个端节点：一个负责发送，另一个负责接收。正如 Maral (2003) 所述，数据的可靠交付意味着数据不发生丢失，且信息内容的至少一个副本能够成功送达目的地。此外，信息内容的交付必须在合理的时延范围内完成，且原始数据的各个部分必须按其原始顺序进行交付。

Network Protocols

实际网络的运行需要借助网络协议，即用于建立并控制网络间信息交换的程序。网络协议可被定义为一套针对联网设备的规则集，它规定了这些设备如何在网络中进行通信以交换信息。这些协议为网络设备提供了一种“通用语言”，使它们能够跨越底层硬件和软件的差异进行交互。正是这种通用语言，确保了设备间的通信能够顺利进行，而无需顾虑其所采用的具体软硬件平台或内部处理机制。一个常用的网络协议实例是地址解析协议（ARP）；该协议负责将动态变化的互联网协议（IP）地址解析（或映射）为固定的物理机器地址（亦称 MAC 地址），反之亦然。这种解析机制之所以必不可少，是因为 MAC 地址与 IP 地址的长度各不相同；若缺乏这种地址转换，设备之间将无法实现有效的通信。例如，IPv4 地址的长度为 32 位，IPv6 地址为 128 位，而 MAC 地址的长度则为 48 位。另一个常见的协议实例是动态主机配置协议（DHCP）；该协议负责为网络终端设备分配 IP 地址，从而使其能够通过 IP 连接与其他设备进行通信。当某台设备首次接入一个部署了 DHCP 服务器的网络时，系统便会自动为其分配一个新的 IP 地址。此外，网络中还应用着许多其他类型的协议（例如用于路由的协议）；在接下来的章节中，我们将对 IP 协议栈及其他相关协议进行更为详尽的探讨。

Network-Level Performance Metrics

对网络性能进行监测与测量是必不可少的，旨在确保网络能够支持预期的应用及其各项需求。网络层面的关键绩效指标（KPI）或服务质量（QoS）指标，是一类用于评估及监测卫星网络性能与可靠性的定量度量标准。主要的 KPI 包括吞吐量、丢包率、时延和抖动。通过对这些指标进行监测与分析，网络运营商能够诊断故障、查找性能瓶颈，并优化相关配置，从而提升网络性能及用户体验。

延迟（或称时延）是指数据在网络中从源端传输至目的端所需耗费的时间。往返时间（RTT）则是指数据从源端出发、抵达目的端后再返回源端所需的总时间。延迟通常以毫秒为单位进行度量。总延迟由以下各项之和构成：(1) 发送端的排队等待时间；(2) 实际数据传输所需的时间（该时间取决于报文长度及所采用的比特率）；(3) 无线链路上的传播延迟（该延迟受卫星轨道高度、所用仰角等因素的影响）；(4) 接收端用于处理数据包接收及生成响应所需的处理时间；以及 (5) 由协议机制（如差错恢复与流量控制）所引发的延迟。在卫星网络中，由于链路距离极长，传播延迟通常在总延迟中占据主导地位。我们在第 5.4.4 节中对传播延迟的相关议题进行了更为详尽的探讨。
