网络基础

网络端口是计算机存储数据的虚拟位置。计算机利用端口来跟踪各种连接和进程；将数据发送到特定端口可以让计算机的操作系统识别相关进程。

Internet Protocol Security (IPsec) 主要使用两个端口来建立安全通道：UDP 端口 4500 用于涉及网络地址转换 (NAT) 的场景，UDP 端口 500 用于连接建立和密钥协商。

您可以使用 NAt-T（需要 UDP 端口 4500）来代替 IPsec（需要 UDP 端口 500 以及 IP 协议 50 和 51）。但是，L2TP 使用 UDP 端口 1701。如果您尝试通过“常规”Wi-Fi 网络发送 IP 流量，并且没有可用的 IPsec 直通选项，则必须打开端口 500 和 4500。

IPsec 使用的端口和协议如下：

  • UDP 端口 500（IKE 用于处理加密密钥）

  • 端口 4500（用于 IPSEC NAT 穿越模式）和协议 UDP

  • 协议 ESP，IPSEC 值 50

  • 协议 AH，IPSEC 值 51

就协议而言，IPsec 对通过基于 IPv4 和 IPv6 的网络传输的数据包进行加密和身份验证。数据包的 IP 报头包含 IPsec 协议报头，这些报头指定了数据包中数据的管理方式，包括如何在网络上路由和发送数据包。IPsec 会在 IP 报头中附加许多元素，例如一种或多种加密方法和安全信息。

IPsec 协议使用一种称为Request for Comments (RFC) 的方法提供网络安全标准的规范。在整个互联网中，RFC 标准用于提供关键信息，帮助开发人员和用户构建、管理和维护网络。
